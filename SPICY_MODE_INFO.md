# Информация о Spicy режиме

## Текущая реализация

Бот использует **spicy режим** для генерации видео через Kie.ai API.

## Важное ограничение

Согласно документации Kie.ai:

> **Note**: When generating videos using external image inputs, Spicy mode is not supported and will automatically switch to Normal.

Это означает:
- ✅ **Spicy режим работает** с изображениями, сгенерированными через Grok на Kie.ai (используя `task_id` + `index`)
- ❌ **Spicy режим НЕ работает** с внешними изображениями (`image_urls`) - автоматически переключается на `normal`

## Текущее поведение бота

Бот отправляет запросы с `mode: "spicy"`, но так как мы используем внешние изображения от пользователей (`image_urls`), Kie.ai автоматически переключит режим на `normal`.

## Как использовать настоящий Spicy режим

Чтобы использовать настоящий spicy режим, нужно:

1. **Сначала сгенерировать изображение через Grok на Kie.ai**:
   - Использовать endpoint `/grok-imagine/text-to-image`
   - Получить `task_id` и `index` сгенерированного изображения

2. **Затем использовать это изображение для видео**:
   - Использовать `task_id` + `index` вместо `image_urls`
   - В этом случае spicy режим будет работать

## Альтернативное решение

Если нужно использовать spicy режим с пользовательскими фотографиями, можно:

1. Сначала обработать фотографию через Grok (улучшить, стилизовать)
2. Получить `task_id` обработанного изображения
3. Использовать это изображение для генерации видео в spicy режиме

## Текущий код

В `services/grok_service.py` установлен `mode: "spicy"`, но для внешних изображений он будет автоматически переключен на `normal` сервером Kie.ai.

## Рекомендация

Если важно использовать именно spicy режим, рассмотрите:
- Двухэтапный процесс: сначала генерация/обработка изображения через Grok, затем видео
- Или использовать `normal` режим явно, так как для внешних изображений spicy все равно не работает
